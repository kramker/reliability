Программа создана для стационарной работы при наличии оператора.
При этом доступна функции сервера, но только на приём запроса.

Работа с программой ведётся через модуль cli.py

Вызовы модуля необходимо осуществлять с параметром new или login

При первой работе с данной программой произведите запуск с параметром new
1. Введите свою почту
2. Дождитесь проверки почты
3. Введите параметр лямбда
4. Введите параметр 1 или 2.  Параметр 1 - эксплутация. Параметр 2 - хранение.
5. Введите параметр 1 или 2.  Параметр 1 - продолжить работу. Параметр 2 - завершение работы.
После вы можете посмотреть полученный данные по пути \reliability\reliability\cli\user в текстовом файле с названием по вашей почте

При продолжении работы с программой произведите запуск с параметром login
1. Введите свою почту
2. Дождитесь проверки почты
3. Введите параметр лямбда 
4. Введите параметр 1 или 2.  Параметр 1 - эксплутация. Параметр 2 - хранение.
5. Введите параметр 1 или 2.  Параметр 1 - продолжить работу. Параметр 2 - завершение работы.
После вы можете посмотреть полученный данные по пути \reliability\reliability\cli\user в текстовом файле с названием по вашей почте

Можно отправить запрос, полученный запрос будет записан по пути reliability\reliability\math\api\request
Проверку запроса можно сделать после запуска server.py по порту localhost:5000
Данные требуется отправить в виде словаря
ключ email значение 'ваш электронный ящик'
ключ lambda значение некое число
ключ exp значение 1 или 2

пример   {'email': 'kramker@gmail.com', 'lambda': 100000, 'exp': 3}

Так же есть протокол тестов
test_api.py - проверит работу api по проверки формата электронной почты
test_local_files - проверит налие папок необходимых для работы программы. Выдаст ошибку с указанием какого пути не хватает, если тест не будет пройден.
